# 🍎 스펠팝 SpellPop — 초등 영단어 스펠링 게임

초등학교 3~6학년을 위한 실시간 웹 기반 영단어 학습 게임입니다.
단어가 잠깐 보였다가 사라지면, 한글 뜻을 보고 **뒤섞인 알파벳 버튼을 순서대로 눌러** 단어를 완성합니다. 빨리 맞힐수록 점수가 높습니다. (카훗과 유사한 실시간 진행)

## 실행 방법

터미널 2개가 필요합니다.

### 1. 백엔드 서버

```bash
cd backend
npm install
npm run dev
```

→ http://localhost:3001 에서 실행됩니다. (`PORT` 환경변수로 변경 가능)

### 2. 프론트엔드

```bash
cd frontend
npm install
npm run dev
```

→ 브라우저에서 http://localhost:5173 접속.
백엔드 주소가 다르면 `frontend/.env`에 `VITE_SERVER_URL=http://주소:포트`를 설정하세요.

## 게임 루프 자동 테스트

브라우저 없이 소켓 이벤트만으로 게임 전체 흐름을 검증하는 스크립트가 있습니다 ([backend/test/simulate.cjs](backend/test/simulate.cjs), 69개 검증).
코드를 수정한 뒤 회귀 여부를 빠르게 확인할 때 사용하세요.

```bash
# 터미널 1: 서버 실행
cd backend && npm run dev

# 터미널 2: 테스트 실행 (기본 localhost:3001 대상)
cd backend && npm run test:sim

# 배포된 라이브 서버 대상으로 검증하려면
TEST_URL=https://spellpop.onrender.com npm run test:sim
```

## 게임 진행 규칙

1. **단어 노출**: 영어 단어가 설정된 시간(1~10초) 동안 표시됩니다.
2. **풀이**: 단어가 사라지고 한글 뜻만 남습니다. 뒤섞인 알파벳 버튼을 순서대로 눌러 단어를 완성하세요. 글자 수를 다 채우면 자동 채점됩니다. (한 글자 지우기 / 전체 지우기 / 패스 가능)
3. **힌트**: 단어가 기억나지 않으면 "💡 힌트 보기"로 정답 단어를 다시 볼 수 있습니다(토글, 열람 시간 무제한). 힌트가 열려 있는 동안에는 답을 입력할 수 없고, 한 번이라도 보면 **그 문제 점수가 절반**이 됩니다. 제한시간은 그대로 흐릅니다.
4. **점수**: 정답 시 `100점 + 남은시간 비율 × 100점`. 고급 ×1.2, 최고급 ×1.5 가중치, 힌트 사용 시 ×½. 오답/패스는 0점.
5. **다음 문제**: 전원이 제출(또는 패스)하면 즉시, 아니면 제한시간이 지나면 자동으로 넘어갑니다.
6. 모든 문제가 끝나면 **시상대 랭킹**(점수·정답률)이 표시됩니다.

## 주요 기능

- **싱글플레이**: 대기실 없이 바로 시작, 본인이 제출하면 즉시 다음 문제
- **멀티플레이**: 6자리 방 코드로 최대 30명 동시 접속, 방장만 시작 가능, 실시간 제출 현황(N/M명) 표시
- **게임 설정** (로비에서 방장/본인이 설정): 난이도 5단계(연습·기본·중급·고급·최고급), 단어 노출 시간 1~10초, 풀이 제한 시간 5~30초, 문제 수 5~30개, 오답 알파벳 1~2개 섞기(고급/최고급 옵션)
- **단어 데이터**: 총 220단어 (연습~고급 각 30개, 최고급 100개 — 초6 심화부터 고3 수능 수준까지) — [backend/src/data/words.ts](backend/src/data/words.ts)에서 쉽게 추가/수정 가능
- **재접속 지원**: 게임 중 새로고침하거나 잠깐 연결이 끊겨도 같은 탭에서 자동으로 재입장해 진행 중인 라운드로 복귀
- **방장 위임**: 방장이 나가면 다른 참가자에게 자동 위임
- **자동 정리**: 빈 방(30초), 방치된 대기방(10분), 종료 후 방치(5분), 전원 이탈한 게임방(2분)을 1분 주기로 스캔해 삭제
- **부정행위 방지**: 정답 단어·타인의 답안은 클라이언트로 전송하지 않으며, 풀이 시간은 서버 시계로 측정하고, 한 문제당 한 번만 제출할 수 있습니다. 알파벳 배치는 서버에서 한 번만 섞어 전원에게 동일하게 제공됩니다.

## 기술 스택

- **프론트엔드**: React 19, Vite, Tailwind CSS v4, Zustand, socket.io-client
- **백엔드**: Node.js, Express, Socket.io, TypeScript (방 상태는 인메모리 `Map<roomId, RoomState>`)

## 배포 (Render 단일 서비스)

배포 빌드에서는 Express 서버 하나가 Socket.io와 빌드된 프론트엔드를 **같은 주소에서** 함께 서빙합니다 (CORS 설정 불필요).

1. 이 저장소를 GitHub에 푸시
2. [dashboard.render.com](https://dashboard.render.com) → **New +** → **Blueprint** → 이 저장소 선택 ([render.yaml](render.yaml)이 자동 인식됨)
3. 배포가 끝나면 발급된 `https://spellpop-xxxx.onrender.com` 주소로 접속

참고:
- **무료 플랜은 15분간 접속이 없으면 잠듭니다.** 다음 접속 시 깨어나는 데 30초~1분 걸리니, 수업 시작 전에 미리 한 번 접속해두세요.
- 게임 방 상태는 서버 메모리에 저장되므로 서버가 재시작되면 진행 중이던 방은 초기화됩니다.
- 로컬에서 배포 모드 그대로 확인: 저장소 루트에서 `npm run build && npm start` → http://localhost:3001
